# 第二轮差距分析：设计文档 vs 当前实现

## 一、分析范围与方法

本文基于以下设计文档与当前 49 个 iOS Swift 文件 + 10 个后端 Python 路由文件的逐项对比：

- **产品设计草案** — 核心功能定义与边界
- **页面信息架构 V1** — 页面职责、跨页跳转、状态回流
- **关键页面组件拆解 V1** — 30+ 组件区定义与通用组件
- **通用组件接口定义 V1** — 9 个通用组件的 Props/Protocol
- **整体服务架构 V1** — 四层架构、四条闭环、异步降级
- **已完成的 11 个任务（overview.md）** — 前一轮补全的 32 项缺失

分析维度涵盖：**页面/组件缺失、交互流缺失、后端 API 缺失、状态管理缺失、安全与边界缺失**。

## 三、跨页面交互流缺失

以下是设计文档中定义的关键跨页链路，对照当前实现的差距：

| 链路 | 设计定义 | 当前状态 |
|------|----------|----------|
| **首页 → 消息中心** | 首页右上角消息图标（带未读徽标）→ 消息列表 | ❌ 无此入口，消息只能通过底部 Tab 4 进入 |
| **首页 → 档案设置** | 首页右上角档案图标 → ProfileView | ❌ 无此入口 |
| **首页 → 阶段说明** | FocusCard "查看阶段说明" → 儿童阶段详情页 | ❌ 无阶段详情页 |
| **即时求助 → 咨询准备** | FollowUpActionBar "查看咨询准备建议" → 咨询准备页 | ❌ 无咨询准备页 |
| **周反馈 → 咨询准备** | ConservativePathNote → 咨询准备页 | ❌ 无咨询准备页 |
| **消息 → 消息详情预览** | 周反馈/风险类消息先展开预览再跳转 | ❌ 所有消息直接跳转 |
| **记录页内 → 语音录制 → 转写** | 语音录制 → 上传 → 后端转写 → 自动生成记录 | ❌ 录制完成后无上传/转写流程 |
| **推送通知 → 应用内页面** | APNs 推送 → 打开 App → 深链到目标页 | ❌ 无推送通知处理 |

## 六、总结

| 维度 | 计数 |
|------|------|
| **P0 缺失项** | 6 |
| **P1 缺失项** | 9 |
| **P2 缺失项** | 13 |
| **总计** | **28** |
| **缺失的跨页交互流** | 8 条 |
| **缺失的后端 API** | 10 个 |
| **设计偏差（需调整）** | 1（四 Tab → 三 Tab） |

前一轮 11 个任务成功补全了 32 项功能缺失，使 iOS 客户端的核心主闭环（Plan→Record→Feedback→Next Plan）和 AI 结果结构化渲染基本成立。本轮分析发现的 28 项缺失主要集中在：**场景化 AI 入口优化、页面空态处理、导航架构微调（三 Tab）、安全认证体系、推送通知闭环、以及若干支撑层页面（咨询准备、阶段说明、记录详情、周回顾）**。这些缺失中，P0 的 6 项直接影响新用户首次使用体验和核心交互质量，建议优先处理。
