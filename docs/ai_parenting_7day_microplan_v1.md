# 18—48个月幼儿家长辅助型 AI 产品：7 天微计划模板 V1

## 一、文档目标

本稿承接《观测模型结构稿 V1.1》与《三阶段具体观察项清单 V1》，把已经稳定下来的**主题判断、观察条目与干预焦点**，进一步翻译成一套可直接供家长执行、也可供产品系统生成的 **7 天家庭微计划模板**。它的作用不是替代专业评估，更不是把家庭支持包装成治疗方案，而是帮助家长在真实生活中，以低负担、低屏幕、强亲子互动的方式，把“看见问题”推进到“开始有节奏地支持”。

这份模板解决的是“系统如何把一个重点主题变成一周内可执行的日常支持安排”，而不是“如何一次性解决孩子全部困难”。因此，它必须保持克制：每次只围绕一个**主干预焦点**展开，最多搭配一个辅助焦点；所有动作都优先嵌入吃饭、洗澡、出门、收拾、共读、玩耍等日常场景；所有反馈都优先描述趋势变化，而不对儿童作诊断性判断。

| 本稿解决的问题 | 本稿不解决的问题 |
|---|---|
| 如何把观察结果转成 7 天家庭微计划 | 如何给出临床诊断或正式筛查结论 |
| 每个微计划应包含哪些固定槽位 | 如何替代治疗师制定个别化康复方案 |
| 八类干预焦点如何进入统一模板 | 如何直接进入页面视觉设计与交互细节 |
| 三年龄阶段如何在同一模板内做难度适配 | 如何对每个条目做正式量化评分 |
| 何时继续执行、何时复盘、何时升级或咨询 | 如何以单次事件推断长期发展结论 |

## 二、设计原则与使用边界

7 天微计划必须同时满足三个条件。第一，它要足够短，能够在家庭现实中被执行，而不是额外制造训练负担。第二，它要足够具体，家长看完后能知道“今天在哪个场景做、怎么说、怎么停、看到什么算有效”。第三，它要足够克制，即便系统已经识别出重点关注主题，也不能把一周计划写成持续施压或高密度训练任务。

微计划的本质不是“多做一点”，而是“在更合适的时机，用更合适的方式，重复创造少量高价值互动机会”。因此，V1 模板固定采用 **一个主目标、两个优先场景、每日一次主练习、一次自然嵌入、一次简短复盘** 的节奏，不鼓励同一周内同时推多个目标，也不鼓励在儿童疲劳、生病、作息紊乱或明显退步时继续加码。

| 设计原则 | 具体要求 | 对模板的直接影响 |
|---|---|---|
| **单焦点优先** | 每份计划只围绕 1 个主焦点展开，必要时允许 1 个辅助焦点 | 避免任务堆叠，便于家长坚持 |
| **场景嵌入优先** | 优先选 2—3 个家庭高频场景承载练习 | 不要求额外开辟长时间训练时段 |
| **家长端可执行** | 动作、话术和记录都应由家长主导 | 保持低屏幕、强亲子、线下执行 |
| **小步推进** | 每天只增加一个小变化，不做强刺激升级 | 降低亲子冲突和执行失败率 |
| **复盘优先于加量** | 先看是否出现有效信号，再决定是否加难度 | 防止把“没效果”误当成“练得不够” |
| **非诊断化表达** | 不使用“落后”“异常”“有问题”等定性判断 | 输出保持专业、克制、行动导向 |
| **安全边界前置** | 不适用条件、升级门槛、咨询提示必须写在计划中 | 防止家庭长期停留在不合适训练中 |

## 三、微计划的输入条件与生成前提

7 天微计划不是由单条记录直接触发，而应由**主题聚合结果**触发。系统在生成计划前，至少需要拿到五类输入：年龄阶段、当前重点主题、相关观察项证据、家庭高频场景、家长当前困扰点。若缺少这些信息，计划就容易变成泛化建议，而不是与当前家庭处境真正贴合的支持方案。

在风险分层上，V1 建议把微计划的适用范围明确为两类。其一，是处于**正常波动**但家长希望更主动支持的主题，此时计划以“给机会、看趋势、做轻支持”为主。其二，是进入**重点关注**的主题，此时计划可以更明确地聚焦一个困难环节，并强化记录与复盘。若已进入**建议咨询**，则产品不再继续输出强化训练型计划，而应切换为“稳定支持 + 补充记录 + 咨询准备清单”的保守路径。

| 生成输入 | 最低要求 | 作用 |
|---|---|---|
| **年龄阶段** | 18—24、24—36、36—48 个月三选一 | 决定任务强度、语言复杂度和回合长度 |
| **主主题与主焦点** | 六主题中确定当前重点，并映射到八类焦点之一 | 决定本周计划主轴 |
| **关联观察证据** | 至少 2—3 条近两周内相关记录，或“2 条记录 + 1 次周回顾” | 保证计划不是凭单次事件生成 |
| **优先家庭场景** | 至少 2 个高频场景 | 让计划真正嵌入生活 |
| **家长困扰点** | 明确“最卡住的环节” | 决定示范话术与复盘重点 |
| **风险状态** | 正常波动 / 重点关注 / 建议咨询 | 决定计划语气与边界 |

## 四、统一模板：每份 7 天微计划必须包含的固定槽位

为了让后续产品、运营、AI 生成和人工审核都可以复用同一结构，每份 7 天微计划都应使用统一槽位。统一槽位的目的，不是增加文档感，而是确保任何一份计划都能清楚回答五个问题：**这周要做什么、在哪做、怎么做、做到什么算有效、什么情况下不要继续。**

> 模板定义：**7 天微计划 = 焦点判断 + 场景安排 + 家长动作 + 示范话术 + 复盘信号 + 升级边界。**

| 槽位名称 | 必填内容 | 设计意图 |
|---|---|---|
| **计划标题** | 年龄阶段 + 主焦点 + 一句话目标 | 让家长一眼知道本周重点 |
| **适用对象** | 月龄段、当前重点主题、适用风险层级 | 保证计划与阶段和风险边界一致 |
| **本周家长目标** | 只写 1 个主目标，必要时补 1 个辅助目标 | 避免一周内目标过多 |
| **优先场景** | 2—3 个家庭高频场景 | 保证可执行性 |
| **不适用条件** | 生病、明显退步、家庭高压变化等 | 防止在不合适窗口继续推进 |
| **每日主练习** | 当天最重要的 1 个动作安排 | 保持节奏清晰 |
| **自然嵌入机会** | 同日可顺手完成的 1 次日常嵌入 | 提高完成率 |
| **示范话术** | 2—4 句可直接复用的话 | 降低家长执行门槛 |
| **观察记录点** | 1—3 个当天应留意的信号 | 形成“做了—看见—记下”的闭环 |
| **日终复盘问题** | 1 个简短问题 | 促使家长关注变化而非感受挫败 |
| **周末总结** | 是否继续 / 是否换焦点 / 是否升级 | 让计划能自然进入下周 |
| **升级门槛** | 连续无变化、出现退步、跨场景持续困难等 | 明确何时不能只靠家庭支持 |
| **咨询提示口径** | 专业、克制、行动导向的建议咨询表达 | 保持安全边界 |

## 五、统一七日节奏：从建立机会到周末决策

V1 不建议让 7 天的安排完全自由生成，否则不同计划之间会失去稳定性。更可执行的方式，是固定一条统一节奏：**第 1 天建基线，第 2 天稳动作，第 3 天转第二场景，第 4 天加入孩子主动性，第 5 天提高泛化，第 6 天看照护者一致性，第 7 天做周总结。** 八类焦点只需在这个骨架里替换具体动作与话术即可。

| 天数 | 当日目标 | 标准动作 | 记录重点 |
|---|---|---|---|
| **Day 1** | 建立本周基线 | 在主场景里只做最基础动作，不加难度 | 孩子是否愿意进入互动；成人是否能稳住节奏 |
| **Day 2** | 固定同一动作 | 在同一场景重复 Day 1 的动作，保持话术一致 | 是否比前一天更容易接住；是否仍需大量提示 |
| **Day 3** | 转到第二场景 | 把同一焦点迁移到另一个高频场景 | 是否只能在单一场景成功 |
| **Day 4** | 增加一点主动性 | 从“成人主导”转向“留一点空位给孩子发起” | 是否出现更主动的看、说、轮到自己或自我表达 |
| **Day 5** | 轻度加量但不加压 | 增加一个回合、一个选择或一个过渡步骤 | 是否能在不明显升级冲突的前提下维持 |
| **Day 6** | 看泛化与照护者一致性 | 尝试换照护者、换时间段或换轻微环境 | 方法是否只对单一成人有效 |
| **Day 7** | 做周末总结 | 回看本周最有效时刻、最卡住环节和下周决策 | 是否继续同焦点、降低难度、换焦点或建议咨询 |

## 六、三年龄阶段的适配规则

虽然微计划模板本身保持统一，但不同年龄阶段的动作密度、话术复杂度和成功标准必须不同。18—24 个月更强调**互动回路启动**，24—36 个月更强调**表达与规则在高频情境中的稳定使用**，36—48 个月更强调**跨场景整合、协商和叙事组织**。如果不对阶段做清晰适配，同一个模板很容易在低龄阶段要求过高，在高龄阶段又过于简单。

| 适配维度 | 18—24个月 | 24—36个月 | 36—48个月 |
|---|---|---|---|
| **每日主练习时长** | 3—5 分钟 | 5—8 分钟 | 5—10 分钟 |
| **每天主目标数量** | 1 个 | 1 个 | 1 个主目标 + 1 个轻辅助点 |
| **话术长度** | 单句、关键词重复 | 短句、二选一、简单先后 | 更完整句式、简单解释与协商 |
| **回合要求** | 1—2 个回合即可 | 2—4 个回合 | 3—5 个回合或一个小任务 |
| **任务重点** | 看、回应、表达尝试、模仿、过渡起步 | 选择表达、轮流、等待、两步理解、规则萌芽 | 协商、叙事、同伴规则、自我表达与恢复 |
| **成功信号** | 是否愿意参与并出现尝试 | 是否开始更稳定地使用目标能力 | 是否能在不同场景和不同对象前更整合地使用 |
| **不宜做的事** | 长时间要求、连续追问、复杂规则 | 高压下强逼说话、一次给太多要求 | 把所有社交问题都当作态度问题 |

## 七、家长语言风格与每日任务强度规范

微计划是否能执行，往往不取决于活动本身，而取决于家长在现场是如何说、如何停、如何看待孩子的反应。因此，V1 除了规定动作，也要规定语言风格。系统给出的家长语言应始终坚持四个顺序：**先描述当前情况，再回应孩子意图，再给一个小提示，最后给选择或下一步。** 不建议直接使用命令堆叠、连续追问、否定式贴标签或羞耻式比较。

| 语言规范 | 建议写法 | 不建议写法 |
|---|---|---|
| **先描述再要求** | “你在看车，我们等一下再走。” | “别看了，快点走。” |
| **先回应意图再给选择** | “你还想玩，我们先收这一块，还是先收那一块？” | “不许闹，马上收完。” |
| **句子要短** | “轮到你。”“你想要水？” | “你已经这么大了为什么还不说清楚？” |
| **允许停顿** | 说完后留 2—5 秒观察孩子是否回应 | 连续追问、连续催促 |
| **复盘说变化，不说评语** | “今天在洗澡前预告后，哭闹时间短了一些。” | “今天表现好多了 / 今天还是很差。” |

## 八、八类干预焦点的模板化展开

以下八类焦点并不是八份完全独立的方案，而是八种可填入统一模板的**标准计划骨架**。后续无论系统根据哪个主题触发计划，都应先落到其中一个主焦点，再按年龄阶段和家庭场景做微调。

### 1. 焦点一：等待回应

这一焦点主要用于共同注意、名字反应、表达尝试不足等情境。它的目标不是逼孩子马上说或马上看，而是在日常互动中**稳定地留出回应空位**，让孩子有机会通过眼神、动作、声音或词语参与进来。

| 模板项 | 建议内容 |
|---|---|
| **本周家长目标** | 在高频互动中先停一停，让孩子更常出现看向人、指向、发声或简单回应 |
| **优先场景** | 吃饭前递物、拿玩具、共读指图、出门看见新东西 |
| **Day 1—2** | 先统一家长节奏：说一次、停 2—5 秒、看孩子是否有任何回应，再继续 |
| **Day 3—4** | 把等待迁移到第二场景，并在孩子出现微弱回应时立刻放大反馈 |
| **Day 5—6** | 从“等待一次”增加到“等待两轮”，看孩子能否出现更主动的再一次回应 |
| **Day 7** | 回看本周最容易出现回应的场景，决定是否继续同场景巩固 |
| **示范话术** | “你想要球吗？……球，给你球。”；“我看到你在看车，我们一起看。” |
| **有效信号** | 看向大人次数增多；拉手前更常先看或发声；单次等待后更容易接住互动 |
| **升级门槛** | 连续 2 周在多个场景中很少回应，或原有回应明显减少 |
| **不适用条件** | 孩子处于明显疲劳、高压崩溃或成人已连续催促失控时 |

### 2. 焦点二：选择表达

这一焦点主要用于需求表达、拒绝、求助表达薄弱，但孩子已有一定表达意图的情境。核心不是让孩子“说得标准”，而是帮助其从哭闹、拉扯、直接抢，逐步过渡到**动作、词语或短句表达选择**。

| 模板项 | 建议内容 |
|---|---|
| **本周家长目标** | 让孩子在高需求场景中，更常用动作、词语或短句表达“要什么 / 不要什么 / 需要帮助” |
| **优先场景** | 吃饭、拿零食、选玩具、穿衣、洗澡前后 |
| **Day 1—2** | 只做二选一，不追求说完整句；接受指向、看向、单词或短句 |
| **Day 3—4** | 同样的选择结构迁移到第二个日常场景，保持选项固定、词语稳定 |
| **Day 5—6** | 在孩子想要得更急时，先命名意图，再给选择，不立刻满足 |
| **Day 7** | 判断孩子是“更愿意表达了”，还是“只在很平静时能表达” |
| **示范话术** | “你要苹果还是香蕉？”；“你想自己来，还是我帮一下？” |
| **有效信号** | 大人猜测次数减少；孩子更常用指向、词语或短句表达选择；冲突前出现表达尝试 |
| **升级门槛** | 多场景持续只靠哭闹或拉扯表达，且 2 周内没有更清晰尝试 |
| **不适用条件** | 成人给选项过多、问法过快、孩子明显超负荷时 |

### 3. 焦点三：动作模仿

这一焦点主要用于模仿起步、功能性游戏、动作序列和游戏参与度不足。目标不是让孩子机械复制，而是通过**看见—跟做—再使用**的过程，启动更自然的互动学习回路。

| 模板项 | 建议内容 |
|---|---|
| **本周家长目标** | 让孩子更愿意跟做简单动作、生活动作或游戏脚本 |
| **优先场景** | 儿歌动作、收拾、洗澡、做饭玩具、假装喂娃娃 |
| **Day 1—2** | 先选 1 个简单而夸张的动作，成人示范时放慢节奏，不要求马上跟做 |
| **Day 3—4** | 保留原动作，再加入同主题第二动作，形成短序列 |
| **Day 5—6** | 尝试把模仿迁移到生活脚本，例如“拿杯子—喂娃娃—擦嘴” |
| **Day 7** | 看孩子是只会跟做，还是开始在稍后自发再现 |
| **示范话术** | “拍拍手，轮到你。”；“娃娃也要喝水，我们喂它。” |
| **有效信号** | 跟做次数增加；提示强度下降；动作开始迁移到另一个相似场景 |
| **升级门槛** | 在多次低压示范下仍极少跟看、跟做，或原有模仿明显减少 |
| **不适用条件** | 动作太快、指令过多、成人同时纠正姿势与结果时 |

### 4. 焦点四：轮流支架

这一焦点主要用于短回合互动、等待轮次、合作任务和对话节奏困难。它的核心不是训练“服从”，而是帮助孩子理解**互动是有来有回的**，并通过可预测节奏降低抢回合和离开互动的概率。

| 模板项 | 建议内容 |
|---|---|
| **本周家长目标** | 让孩子在游戏、对话或任务中，能多接住 1—2 个轮次 |
| **优先场景** | 推球、搭积木、轮流收玩具、对话接话、桌面小游戏 |
| **Day 1—2** | 固定使用明显回合词，例如“轮到妈妈”“轮到你” |
| **Day 3—4** | 在第二场景里保持同样回合标记，减少额外语言说明 |
| **Day 5—6** | 轻度增加轮次数或合作步骤，但不增加规则复杂度 |
| **Day 7** | 判断孩子是否能在无过多催促下维持更稳定节奏 |
| **示范话术** | “我一个，你一个。”；“先等我放好，轮到你。” |
| **有效信号** | 离开互动次数减少；抢回合前能短暂等待；对话或动作来回更稳定 |
| **升级门槛** | 多个场景下持续无法接住轮次，且伴随高冲突或明显退步 |
| **不适用条件** | 游戏规则过复杂、等待时间拉得过长、多人同时指挥孩子 |

### 5. 焦点五：过渡预告

这一焦点主要用于转场、等待、停止喜欢活动和任务切换。目标不是让孩子“不闹”，而是通过**提前知道、提前看见、提前准备**，降低进入高压状态的概率。

| 模板项 | 建议内容 |
|---|---|
| **本周家长目标** | 在固定高压场景中，通过预告和先后结构，降低转场时的突然性 |
| **优先场景** | 出门、洗澡、收玩具、睡前、结束屏幕或结束玩耍 |
| **Day 1—2** | 找出 1 个最常失控的转场场景，只做“提前预告 + 先后语言” |
| **Day 3—4** | 在第二个转场场景重复同样结构，话术保持一致 |
| **Day 5—6** | 加入一个可预测步骤，例如倒数、收最后一个玩具、自己按灯 |
| **Day 7** | 比较本周是否更容易从提醒进入动作，而不是直接升级冲突 |
| **示范话术** | “再玩一次，我们就去洗澡。”；“先穿鞋，再出门。” |
| **有效信号** | 听到预告后抗拒时间缩短；需要的身体协助减少；更容易进入下一步 |
| **升级门槛** | 同一高压场景连续两周都强烈升级，或多个场景同时显著恶化 |
| **不适用条件** | 预告太早或太频繁导致失效；当下已进入高强度崩溃状态时继续讲道理 |

### 6. 焦点六：情绪命名

这一焦点主要用于受挫、被拒绝、等待、恢复困难和高压后整理。其核心不是让孩子立刻冷静，而是帮助其在成人共同调节中，逐步把模糊体验变成**可被看见、可被说出、可被收回来的感受**。

| 模板项 | 建议内容 |
|---|---|
| **本周家长目标** | 在孩子卡住或受挫时，先命名感受和意图，再进入下一步动作 |
| **优先场景** | 穿衣失败、玩具拿不到、被拒绝、等待、与同伴冲突后 |
| **Day 1—2** | 只做“你很想…… / 你现在不高兴”这类简短命名，不急着讲规则 |
| **Day 3—4** | 在命名后补一个小选择或一个恢复动作，例如抱一下、坐旁边、换一种做法 |
| **Day 5—6** | 在孩子稍平静后，加入一句简单复盘，帮助其把事件和感受连起来 |
| **Day 7** | 判断有效点是在“更快回稳”，还是“更早出现可被理解的表达” |
| **示范话术** | “你很想自己来，现在卡住了。”；“你生气了，我在这儿，我们慢一点。” |
| **有效信号** | 爆发持续时间缩短；成人命名后更容易接住；孩子开始用动作或词语表达不舒服 |
| **升级门槛** | 高频高强度失控持续影响家庭日常，或恢复明显变慢并伴随退步 |
| **不适用条件** | 成人用情绪命名替代全部边界；在孩子已极度失控时连续追问原因 |

### 7. 焦点七：社交预演

这一焦点主要用于接近同伴、进入活动、排队等待、分享冲突和陌生场景适应。目标不是把孩子迅速推入社交，而是通过**事前预热、场中支架、事后复盘**，让其更容易进入并停留在社交情境里。

| 模板项 | 建议内容 |
|---|---|
| **本周家长目标** | 在一个可控社交场景里，帮助孩子更顺利地进入、停留和退出 |
| **优先场景** | 小区玩耍、见亲友、平行玩、排队、简单规则游戏 |
| **Day 1—2** | 在出发前做一句预演，告诉孩子将看到谁、可能做什么、如果不想可以先怎样 |
| **Day 3—4** | 现场只支持一个小目标，例如“先靠近看一会儿”或“轮到时递一次” |
| **Day 5—6** | 在第二个相似社交场景重复相同脚本，看是否更容易进入 |
| **Day 7** | 复盘孩子是更容易靠近了，还是更容易在冲突后回到活动中 |
| **示范话术** | “等会儿会有小朋友在玩，你可以先看，再决定要不要过去。”；“轮到你递一下球。” |
| **有效信号** | 进入活动速度变快；停留时间稍长；冲突后更容易在成人帮助下回到规则中 |
| **升级门槛** | 长期多场景明显回避或冲突升级，且家庭支持后仍持续加重 |
| **不适用条件** | 把社交预演变成强逼参与；场景本身过于拥挤、噪声过大或要求明显超龄 |

### 8. 焦点八：叙事支架

这一焦点主要用于假装游戏、顺序表达、讲述刚发生的事和角色情节延续。它的目标不是追求完整讲故事，而是帮助孩子把**人、物、动作、结果**更自然地连起来，并逐步形成可分享的经验组织。

| 模板项 | 建议内容 |
|---|---|
| **本周家长目标** | 让孩子在游戏或经历后，更常把“谁 / 做了什么 / 结果怎样”连起来 |
| **优先场景** | 共读、假装游戏、晚间回顾、外出回来后、图片回顾 |
| **Day 1—2** | 成人先提供最短脚手架，只补“谁在做什么” |
| **Day 3—4** | 加入顺序词或结果词，例如“先……然后……”或“后来……” |
| **Day 5—6** | 把脚手架迁移到另一种材料，如玩具、图片或当天经历 |
| **Day 7** | 看孩子是更会接续成人的话，还是开始主动补充一小段 |
| **示范话术** | “小熊在吃饭。”；“先去公园，然后坐滑梯。” |
| **有效信号** | 假装情节更连贯；讲述不再只剩单词堆积；更愿意重复分享刚发生的事 |
| **升级门槛** | 36—48 个月阶段在多场景下持续很难组织简单顺序与经验分享，且伴随其他主题明显偏弱 |
| **不适用条件** | 在孩子明显不想讲、已疲劳或现场要求太抽象时强行追问 |

## 九、周复盘规则：何时继续、何时换焦点、何时升级

7 天微计划的价值，不在于完成 7 天本身，而在于它能否帮助家长和系统做出更稳妥的下一步决策。因此，周末复盘必须是模板中的强制环节。V1 建议复盘只回答三个问题：**本周哪个场景最有效、哪个环节最卡住、下周应继续 / 降难度 / 换焦点 / 建议咨询。**

| 周末结论 | 进入条件 | 下一步动作 |
|---|---|---|
| **继续同焦点** | 已出现轻度积极信号，且家长可持续执行 | 保持原焦点，再巩固 1 周 |
| **继续但降难度** | 计划方向正确，但任务强度偏高，孩子和家长都吃力 | 缩短时长、减少回合、减少场景数 |
| **换同主题下的辅助焦点** | 原焦点没有明显效果，但主题判断仍成立 | 例如从“等待回应”换到“选择表达” |
| **换主题** | 复盘发现主要困难并不在原主题上 | 重新聚合观察证据后再生成计划 |
| **建议补充观察** | 记录量不足、场景过窄或照护者口径冲突 | 先补一周记录，再决定计划 |
| **建议咨询** | 出现退步、多主题并存、持续无改善或家庭困扰显著升高 | 切换为咨询准备与稳定支持路径 |

## 十、产品承接建议：微计划的数据结构与生成约束

为了保证后续系统实现时不会把微计划做成自由写作模块，V1 建议在产品层面提前固定关键字段。这样既便于规则引擎先行，也便于后续 AI 只在受控槽位内生成更贴合家庭语境的话术和说明。

| 字段 | 类型 | 说明 |
|---|---|---|
| `plan_id` | 字符串 | 计划唯一标识 |
| `stage_band` | 枚举 | 18—24 / 24—36 / 36—48 |
| `primary_theme` | 枚举 | 六主题之一 |
| `focus_tag` | 枚举 | 八类干预焦点之一 |
| `risk_level` | 枚举 | 正常波动 / 重点关注 / 建议咨询 |
| `linked_observation_ids` | 数组 | 关联观察项或关键事件记录 |
| `primary_goal` | 文本 | 本周唯一主目标 |
| `priority_scenes` | 数组 | 2—3 个优先场景 |
| `not_applicable_conditions` | 数组 | 不适用条件 |
| `daily_steps` | 数组 | 7 天标准结构化安排 |
| `sample_scripts` | 数组 | 2—4 句示范话术 |
| `review_signals` | 数组 | 复盘信号 |
| `escalation_threshold` | 数组 | 升级门槛 |
| `consultation_copy` | 文本 | 建议咨询时的克制话术 |

## 十一、结论与下一步

《7 天微计划模板 V1》的核心价值，在于把此前已经稳定下来的**主题判断、观察项证据、干预焦点与风险边界**，进一步收敛成一套**可生成、可执行、可复盘**的家庭支持结构。到这一层为止，产品已经不再停留于“观察定义”，而开始具备“把观察翻译成一周行动”的能力。

在此基础上，后续最自然的推进顺序已经更清晰。第一，可以继续将八类焦点中的 1—2 类扩写为**完整示范周计划样稿**，用于验证模板是否足够易懂。第二，可以把“7 天微计划—周反馈—即时求助”串成一条**页面信息架构**，明确家长在产品里的使用路径。第三，可以补一版 **建议咨询场景下的稳定支持与就诊准备模板**，使风险升级链路完整闭环。
